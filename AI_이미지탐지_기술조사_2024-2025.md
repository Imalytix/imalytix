## 1. AI 생성 이미지 탐지 (AI-Generated Image Detection)

> 핵심 키워드: Diffusion model artifact, GAN fingerprint, deepfake detection

| # | 논문명 / 기술 | 저자 / 발표 | 핵심 아이디어 | 성능 / 정확도 | 한계점 | 구현 가능성 |
|---|---|---|---|---|---|---|
| 1 | **Towards Universal Fake Image Detectors that Generalize Across Generative Models** | Ojha et al., CVPR 2023 (Wisconsin AI Vision) | CLIP 임베딩 공간에서 Nearest Neighbor 분류 → 훈련 없이도 새로운 생성 모델에 범용 적용 가능 | GAN/Diffusion 혼합 벤치마크에서 기존 CNN 탐지기 대비 일반화 성능 우위 | 고해상도·영상에는 미검증 | ★★★★★ 오픈소스(GitHub: WisconsinAIVision/UniversalFakeDetect), 추가 학습 불필요 |
| 2 | **AEROBLADE: Training-Free Detection of Latent Diffusion Images Using Autoencoder Reconstruction Error** | Ricker et al., CVPR 2024 | LDM의 오토인코더가 **실제 이미지보다 생성 이미지를 더 정확히 복원**한다는 특성을 이용, 재구성 오차로 탐지 | Stable Diffusion·Midjourney 대상으로 학습 기반 탐지기에 근접한 성능 | Dall-E 3 등 비LDM 모델에는 효과 제한적 | ★★★★★ 오픈소스(GitHub), 학습 데이터 불필요 |
| 3 | **Level Up the Deepfake Detection: GAN vs Diffusion Hierarchical Classification** | Corvi et al., arXiv 2303.00608 (2023) | 9종 GAN + 4종 Diffusion 데이터 수집 → Real vs AI / GAN vs DM / 아키텍처 분류 3단계 계층적 탐지 | 평균 정확도 94.42% (다운샘플링 제거 변형), GAN→DM 재학습으로 AUROC 대폭 개선 | 새 모델 출시 시 재학습 필요 | ★★★☆☆ 구현 가능하나 다종 학습 데이터 수집 필요 |
| 4 | **Artifact Feature Purification for Cross-domain Detection of AI-generated Images** | arXiv 2403.11172 (2024) | 도메인 간 아티팩트 특징을 정제(purification)하여 학습, 새 생성 모델에 대한 일반화 강화 | 크로스 도메인 탐지에서 기존 방법 대비 성능 향상 | 확인 필요 (상세 수치 미공개) | ★★★☆☆ 오픈소스 여부 확인 필요 |
| 5 | **Detecting AI-Generated Images via Diffusion Snap-Back Reconstruction** | arXiv 2511.00352 (2025) | 다양한 강도의 노이즈 추가 후 역확산(reverse diffusion) 복원 다이나믹스를 분석해 생성 이미지 판별 | 멀티-스트렝스 재구성으로 기존 탐지기 대비 범용성 향상 | 추론 시간 오래 걸림(다수 역확산 필요) | ★★★☆☆ 방법론 공개, 구현 복잡도 높음 |

### 영역 1 소결
- **가장 실용적 접근**: AEROBLADE(학습 불필요) + UniversalFakeDetect(CLIP 기반 범용)
- 2025년 트렌드: 단순 이진 분류 → **모델 출처 추적(attribution)** 및 **멀티모달 탐지**로 진화 중

## 2. 이미지 위변조 탐지 (Image Forgery / Manipulation Detection)

> 핵심 키워드: Splicing, Copy-Move, Inpainting, 주파수 도메인 분석

| # | 논문명 / 기술 | 저자 / 발표 | 핵심 아이디어 | 성능 / 정확도 | 한계점 | 구현 가능성 |
|---|---|---|---|---|---|---|
| 1 | **TruFor: Leveraging All-Round Clues for Trustworthy Image Forgery Detection and Localization** | Guillaro et al., CVPR 2023 (UNINA) | CLIP 사전학습 ViT-L 백본 + NoisePrint++ (카메라 핑거프린트 추출기)를 Transformer 디코더로 융합, 픽셀 레벨 위변조 맵 + 이미지 레벨 신뢰도 점수 동시 출력 | CASIAv1: F1=0.789, Pixel-AUC=0.946 (CVPR 2023 기준 SOTA) | 학습 데이터가 실제 카메라 이미지 필요, 인페인팅에 다소 약함 | ★★★★★ 오픈소스(GitHub: grip-unina/TruFor), PyTorch 구현 제공 |
| 2 | **IML-ViT: Benchmarking Image Manipulation Localization by Vision Transformer** | arXiv 2307.14863 (2023) | 순수 ViT 백본을 IML(Image Manipulation Localization) 태스크에 최초 적용, 기존 CNN 기반 방법의 벤치마크 기준선 정립 | 기존 SOTA 모델 전체 상회, 복수 데이터셋 통일 평가 프레임워크 제시 | 고해상도 이미지에서 연산 비용 큼 | ★★★★☆ 오픈소스, ViT 백본 파인튜닝 가능 |
| 3 | **FakeShield: Explainable Image Forgery Detection and Localization via Multi-modal LLM** | Xu et al., ICLR 2025 | GPT-4o로 생성한 image-mask-description 트리플렛(MMTD-Set)으로 MLLM 파인튜닝, 픽셀 레벨 위치 탐지 + **자연어 설명 동시 제공** | IMD2020에서 MVSS-Net 대비 ACC +0.08/F1 +0.05; DeepFake 탐지 0.93, AIGC-Editing 탐지 0.98 | GPT-4o 의존 데이터 구축 비용; 추론 속도 느림 | ★★★☆☆ 오픈소스(GitHub: zhipeixu/FakeShield), 학습 인프라 필요 |
| 4 | **SAMIF: Adapting Segment Anything Model for Image Inpainting Forensics** | ACCV 2024 | Meta SAM(Segment Anything)을 인페인팅 탐지에 특화 파인튜닝, 인페인팅 영역의 경계 아티팩트를 마스크로 정밀 세그멘테이션 | ACCV 2024 발표, 인페인팅 탐지 세부 성능 우위 | 일반 splicing/copy-move 탐지는 별도 모델 필요 | ★★★★☆ SAM 기반이라 파인튜닝 접근 가능 |
| 5 | **Frequency-driven Deep Learning Network for Image Splicing Forgery Detection** | ScienceDirect / Engineering Applications of AI, 2024 | FFT/DCT 주파수 스펙트럼 분석 + CNN 이중 브랜치(공간 + 주파수) 융합, Splicing 경계의 주파수 불일치 특성 활용 | 주파수 도메인 특화로 압축·후처리에 강건한 탐지 | 훈련 시 고품질 위변조 데이터셋 필요 | ★★★☆☆ 방법론 구현 가능, 논문 코드 공개 여부 확인 필요 |

### 영역 2 소결
- **즉시 활용 가능**: TruFor (오픈소스, 현재 기준 general-purpose SOTA)
- 2024~2025 트렌드: SAM·MLLM 등 대형 비전 파운데이션 모델을 위변조 탐지에 적응(PEFT) 하는 방향으로 빠르게 이동 중

## 3. 이미지 출처 및 신뢰성 검증 (Image Provenance & Authenticity)

> 핵심 키워드: C2PA, Content Credentials, Watermarking, Perceptual Hashing

| # | 논문명 / 기술 | 저자 / 발표 | 핵심 아이디어 | 성능 / 정확도 | 한계점 | 구현 가능성 |
|---|---|---|---|---|---|---|
| 1 | **C2PA (Coalition for Content Provenance and Authenticity) 표준** | Adobe, Microsoft, Google, Sony 등 공동 (v1.3: 2023.01 / v2.0: 2023.11 / v2.1: 2024.09 / v2.2: 2025.05) | 이미지에 암호화 서명된 Manifest(생성·편집 이력)를 바인딩, 소프트 바인딩(워터마크 연동) + 하드 바인딩(해시) 이중 구조 | 라이카 M11-P(2023)에 세계 최초 카메라 탑재, 업계 표준으로 급속 확산 | 이미지 포맷 변환·리사이징 시 하드 바인딩 끊김; 민간 도입률 아직 낮음 | ★★★☆☆ 오픈 표준이지만 라이브러리 통합 필요(c2pa-rs, c2pa-node 등) |
| 2 | **The Stable Signature: Rooting Watermarks in Latent Diffusion Models** | Fernandez et al. (Meta AI), ICCV 2023 | LDM 디코더를 바이너리 시그니처 조건으로 파인튜닝, 모든 생성 이미지에 불가시 워터마크 임베딩 → 사전학습 워터마크 추출기로 추출 | 10% 크롭 후에도 90%+ 탐지율 (FPR < 1e-6), 다양한 후처리에 강건 | LDM 모델 자체를 파인튜닝 해야 함 (서비스 운영사 협조 필요) | ★★★★☆ 오픈소스(GitHub: facebookresearch/stable_signature) |
| 3 | **InvisMark: Invisible and Robust Watermarking for AI-generated Image Provenance** | Microsoft Research, WACV 2025 (arXiv 2411.07795) | 인코더-디코더 구조로 AI 생성 이미지에 불가시 워터마크 삽입, JPEG 압축·스크린샷·리사이징에도 강건 | WACV 2025 발표, 다양한 공격에 대한 강건성 벤치마크 제시 | 외부 공격(생성 후 이미지 편집)에 대한 완전한 방어는 어려움 | ★★★★★ 오픈소스(GitHub: microsoft/InvisMark) |
| 4 | **Tree-Ring Watermarks: Fingerprints for Diffusion Images that are Invisible and Robust** | Wen et al., arXiv 2305.20030 (NeurIPS 2023) | Diffusion 역확산의 초기 노이즈 벡터에 동심원(tree-ring) 패턴을 심어 탐지, 추출 시 역확산 후 주파수 도메인 분석 | 이미지 편집·증강에 매우 강건, 탐지 정확도 99%+ 보고 | Diffusion 모델의 역확산 과정 접근 필요 (블랙박스 서비스에 적용 어려움) | ★★★★☆ 개념 공개, 오픈소스 구현체 존재 |
| 5 | **Provenance Detection for AI-Generated Images: Combining Perceptual Hashing, Homomorphic Encryption, and AI Detection Models** | arXiv 2503.11195 (2025) | pHash·PDQ 등 퍼셉추얼 해싱 + 동형암호 기반 프라이버시 보존 + AI 탐지 모델을 결합한 통합 출처 검증 프레임워크 | 실용적 파이프라인 제안, 성능 수치는 확인 필요 | 동형암호 연산 오버헤드 큼; 실시간 서비스 적용 어려움 | ★★☆☆☆ 연구 단계, 구현 복잡도 매우 높음 |

### 영역 3 소결
- **즉시 활용 가능**: Stable Signature + InvisMark (오픈소스 워터마킹)
- C2PA는 업계 표준이 되고 있으므로 서비스 호환성 측면에서 필수 검토 항목
- Tree-Ring은 연구적으로 가장 강건하나 Diffusion 파이프라인 접근 전제

## 스타트업 차별화 포인트 요약

> "독자 기술로 가져갈 수 있는 포인트" — 3~5가지

1. **탐지 + 워터마킹 + 출처 추적의 통합 파이프라인**: 현재 연구는 각 영역이 분절되어 있음. "이미지 생성 시 워터마크 삽입(Stable Signature/InvisMark) → 유통 중 위변조 탐지(TruFor) → 출처 증명(C2PA 연동)"을 단일 SDK로 제공하는 엔드투엔드 솔루션은 아직 공백 상태.

2. **Training-Free 경량 탐지기의 상용화**: AEROBLADE·UniversalFakeDetect는 학습 없이도 높은 성능을 보이지만 API 서비스화된 제품은 거의 없음. 이를 SaaS/API로 포장하면 재학습 비용 없이 새 생성 모델 대응 가능한 차별화 포지셔닝 가능.

3. **설명 가능한(Explainable) 위변조 탐지**: FakeShield처럼 "어디가, 왜 조작되었는지" 자연어로 설명하는 기능은 법적·미디어 팩트체킹 시장에서 수요가 높음. MLLM 기반 파인튜닝으로 한국어 설명까지 추가하면 국내 시장 특화 가능.

4. **C2PA 생태계 내 퍼셉추얼 해싱 강화**: C2PA의 소프트 바인딩(이미지 변환 후 Manifest 재연결)은 아직 표준 완성도가 낮음. pHash/PDQ 기반의 강건한 소프트 바인딩 모듈을 오픈소스로 기여 + 상용 지원 모델은 커뮤니티·기업 이중 시장 공략 가능.

5. **카메라/디바이스 핑거프린트 + 출처 연동**: TruFor의 NoisePrint++ 방식처럼 카메라 센서 노이즈로 촬영 기기를 식별하는 기술을 C2PA 하드 바인딩에 결합하면, 스마트폰 업체나 보안 카메라 업체와의 B2B 파트너십으로 연결 가능.
