# 기능 확인용 데이터셋 구축 가이드

## 목적

Imalytix의 기능을 효과적으로 검증하기 위해, 실제 배포 전에 로컬에서 재현 가능한 평가셋을 구성합니다.

이 데이터셋은 다음 기능을 확인하는 데 사용합니다.

- 이미지 유사도 검색
- 도용 흔적 탐지
- 위조/편집 흔적 탐지
- source attribution
- trust score

## 권장 구성

- 총 용량: 5GB 이하
- 이미지 수: 200~1000장 정도부터 시작
- 라벨:
  - `REAL`
  - `AI`
  - `EDITED`
  - `REPOST`
- 카테고리:
  - 인물
  - 제품
  - 풍경
  - 음식
  - 인테리어
  - 텍스트 포함 이미지

## 구축 방법

1. 이미지를 먼저 수집한다.
2. 파일명 또는 폴더명에 라벨을 넣는다.
3. 같은 카테고리 안에서도 다양한 압축/리사이즈/캡처본을 섞는다.
4. 원본과 재업로드본을 함께 넣는다.
5. AI 이미지와 실제 이미지를 같은 테마로 쌍을 맞춘다.
6. manifest를 남긴다.

## 왜 이렇게 구성하나

- pHash는 거의 같은 이미지를 잘 잡는다.
- embedding은 의미상 비슷한 이미지를 잡는다.
- source attribution은 파일명/URL/도메인 단서를 잡는다.
- 위조 탐지는 편집 흔적을 잡는다.

즉, 한 가지 유형만 넣으면 시스템이 편향되므로, 여러 유형을 섞어야 성능과 오탐을 함께 볼 수 있다.

## 자동 생성 스크립트

아래 스크립트는 로컬 폴더에서 이미지를 수집해서 5GB 제한 안에서 복사하고 manifest를 생성합니다.

```bash
python scripts/build_validation_dataset.py \
  --source samples_test \
  --output dataset/validation \
  --max-size-gb 5 \
  --per-category-limit 30
```

생성 결과:

- `dataset/validation/manifest.csv`
- `dataset/validation/summary.txt`
- 카테고리별 이미지 복사본

## 현재 단계에서의 권장 전략

- 1차: `samples_test` 기반 소형 벤치마크
- 2차: 실제 거래 플랫폼, 블로그, SNS 유사 포맷 샘플 추가
- 3차: 실제 도용/재업로드 사례를 별도 `positive` 집합으로 관리

## 주의사항

- 개인정보가 포함된 이미지는 반드시 익명화 또는 테스트 전용으로만 사용해야 합니다.
- 출처가 불명확한 이미지를 무단으로 대규모 저장하는 방식은 피해야 합니다.
- 플랫폼 약관과 저작권 이슈를 확인해야 합니다.
