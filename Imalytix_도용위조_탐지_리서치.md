## 2. 테스트를 어떻게 하는 것이 좋은가

### 권장 순서

1. 단위 테스트부터 실행한다.
2. API 테스트로 요청/응답 구조를 확인한다.
3. 실제 서버를 띄워 헬스 체크를 확인한다.
4. 샘플 이미지를 넣어 pHash, 메타데이터, trust score 흐름을 본다.
5. 캐시 재사용과 유사 이미지 처리까지 확인한다.

### 권장 테스트 항목

- 이미지 업로드 성공 여부
- 이미지 URL 보안 차단 여부
- pHash 캐시 재사용 여부
- 메타데이터 기반 AI 흔적 탐지 여부
- 도용 위험 점수 계산 여부
- 위조 위험 점수 계산 여부
- 최종 trust score / risk score 범위 보정 여부

---

## 3. 이미지 도용 탐지의 원리

### 현재 프로젝트의 판단 기준

- 동일 pHash 중복 저장 여부
- 유사 pHash 거리
- source URL 패턴
- 파일명 / 경로 패턴
- 메타데이터 내 복제 / 재업로드 신호

### 핵심 아이디어

- 같은 이미지는 pHash가 거의 동일하다.
- 약간 편집된 이미지도 pHash 거리로 어느 정도 포착할 수 있다.
- 출처 URL이나 파일명 패턴이 반복/복제 성격이면 도용 가능성을 올린다.

### 장점

- 빠르다.
- 서버 비용이 낮다.
- 동일 이미지 재사용에는 매우 효과적이다.

### 한계

- 크롭, 회전, 리사이즈, 재압축에 약하다.
- 부분 편집이나 합성에는 취약할 수 있다.
- 원본 출처가 없으면 확정 판정이 어렵다.

---

## 4. 이미지 위조 탐지의 원리

### 현재 프로젝트의 판단 기준

- EXIF / PNG metadata
- Stable Diffusion, ComfyUI, Midjourney 등 AI 도구 흔적
- 전용 AI detector 결과
- Vision LLM의 시각적 이상 근거
- 출처 URL 패턴

### 핵심 아이디어

- AI 생성 도구는 종종 메타데이터 흔적을 남긴다.
- Vision 모델은 손상, 반복 패턴, 조명 불일치 같은 이상 징후를 읽어낸다.
- 전용 detector는 “AI 생성 확률”을 수치로 제공한다.

### 장점

- 설명 가능한 근거를 만들기 쉽다.
- 메타데이터와 시각 근거를 함께 볼 수 있다.
- 사람에게 결과를 납득시키는 데 유리하다.

### 한계

- 메타데이터는 삭제되기 쉽다.
- Vision 모델은 저해상도, 후처리, 압축 노이즈에 흔들릴 수 있다.
- 전용 detector도 모델별 편향이 있다.

---

## 5. 신뢰성 스코어링의 구조

### 현재 계산 축

- 도용 위험
- 위조 위험
- 최종 trust score

### 개념

- 도용 위험이 높으면 “이미지 자체가 재사용/복제되었을 가능성”이 커진다.
- 위조 위험이 높으면 “이미지 자체가 인위적으로 조작되었을 가능성”이 커진다.
- 둘을 합쳐 최종적으로 “이 이미지를 얼마나 믿을 수 있는가”를 본다.

### 현재 반영 방식

- 도용 위험과 위조 위험을 가중 평균으로 합산
- `trust_score = 100 - combined_risk`
- `trust_analysis` 응답에 신뢰 가능 여부와 증거를 함께 반환

## 6. 문제점과 보완점

### 도용 탐지의 문제점

- pHash는 이미지가 조금만 바뀌어도 약해질 수 있다.
- 동일 이미지인지, 파생 이미지인지 구분이 어렵다.
- 원본 출처 확인 없이 단정하기 어렵다.

### 위조 탐지의 문제점

- 메타데이터가 삭제되면 근거가 약해진다.
- LLM/Vision 결과는 확률적이며 오판 가능성이 있다.
- 실제 사진도 보정이 심하면 AI처럼 보일 수 있다.

### 앞으로 보완할 점

- 유사도 기반 클러스터링 강화
- 크롭 / 회전 / 재압축에 강한 특징 추출 추가
- C2PA / Content Credentials 연동
- OCR 기반 텍스트 변조 탐지
- 워터마크 / 로고 / 출처 표기 탐지
- 원본 출처 DB 축적
